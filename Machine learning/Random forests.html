Random forests

  • Ensemble of decision trees which can be used for classification or regression

Methodology

  • For i = 1,...,B:
    • Choose bootstrap sample D_i from D.
    • Construct tree T_i using D BUT at each node choose a random subset of m features, where m is less than or equal to the number of total features. Only consider splitting on these features at the given node.
    • For the split on each attribute, we use information gain to find a best split:
    • In a classification task, we take a majority vote (i.e. more trees ended with this result than the other). For regression, we average predictions.

Prediction

  • We need to find the leaf node posteriors p_t(r|u).
    • In classification, we end up with a histogram at each leaf node, so we count the number of training examples in each class to estimate p_t(r|u)
    • In regression, our result is a density which we assume is a Gaussian. We can calculate mean, variance of each class in the training examples of the node, where r is the label and u the data.
  • Posterior probability:

That is, the average of the posterior probability for each leaf node.

  • The prediction is then given by a decision rule:

That is, the r with the highest probability given the data.

Tidbits

  • More trees ‚Üí less variance ‚Üí better performance
  • The trees that go into a forest are basically entirely random - its an ensemble method which will contain good and bad trees.
  • Hyperparameter could be minimum number of examples in a leaf, or maximum depth of tree
  • Can run a grid search on n (number of examples on a leaf) and tau (split threshold). You have to run this many times on each node.
