K2 Algorithm

  • We need to learn these Bayesian network structures from the data. The problem is, we can't check every single structure, especially higher numbers of variables, as the number of possible DAGs increases super-exponentially. So we use a scoring algorithm to search for a representative structure.
  • A scoring algorithm allows us to start from an initial structure and move to a neighbour with better scores. We can end up at a local maximum of the score - global maxima not guaranteed
  • The k2 algorithm:
    • Applies to discrete variables: x1, ..., xn
    • Assumes each node can have a maximum of n parents (set by us)
    • We start with an initial Bayesian network (e.g. a head to tail) and add parents incrementally to each node until we reach the local maximum
    • Assumes an order of the variables (e.g. x1 ‚Üí x2 ‚Üí x3 ‚Üí x4)
    • Greedy search means we don't backtrack ‚Üí if there's no neighbours with a higher score, that's the end of the algorithm. However, we can restart with a different initial structure and see if there's any difference.

  • The scoring function evaluates how representative a Bayesian network is of the data.
